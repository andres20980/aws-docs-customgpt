# Contribuyendo al AWS Docs Markdown Project

## Cómo contribuir
Estamos encantados de recibir contribuciones de la comunidad. Aquí hay algunas formas en que puedes contribuir:

- Reportando bugs.
- Sugerir mejoras.
- Añadiendo nuevas funcionalidades.
- Mejorando la documentación.

## Proceso
1. **Fork el repositorio**: Haga un 'fork' del proyecto en su cuenta de GitHub.
2. **Clone su fork**: `git clone https://github.com/tu-usuario/aws-docs-markdown.git`
3. **Cree una rama**: `git checkout -b mi-nueva-caracteristica`
4. **Haga sus cambios**: Asegúrese de probar su código y que cumpla con las guías de estilo del proyecto.
5. **Commit sus cambios**: `git commit -am 'Añadir alguna característica'`
6. **Push a la rama**: `git push origin mi-nueva-caracteristica`
7. **Solicite un pull request**: Vaya a su fork en GitHub y cree una nueva pull request.

## Guías de estilo de código
- Utilice indentaciones de 4 espacios para Python.
- Siga las [PEP8](https://www.python.org/dev/peps/pep-0008/) guías de estilo de Python.
- Escriba comentarios claros y concisos en su código cuando sea necesario.

## Pruebas
Asegúrese de añadir pruebas para nuevas funcionalidades o cambios en el código. Verifique que todas las pruebas pasen antes de enviar cualquier pull request.

Gracias por contribuir a mejorar este proyecto!
